# SegWitz Communications Hub (SCH) – Project Prompt

This file is the master project prompt for SCH. The full technical build specification (modules, database, auth, Brevo, templates, deployment, etc.) is implemented in this codebase. See `README.md` and `DEPLOYMENT.md` for technical details.

---

# IMPORTANT PROJECT EXECUTION RULES

You are NOT allowed to dump large amounts of code and leave me guessing what to do next.

I am a non-technical user and I need extremely clear guidance.

For every task you complete, follow this exact format:

## PHASE X COMPLETED

Explain in simple language:

* What was built
* Why it was built
* Which files were created
* Which files were modified
* Which database tables were added
* Which environment variables are required

---

## WHAT I NEED TO DO NOW

Give me ONLY the next action.

Example:

Step 1:
Create a Supabase project.

Detailed instructions:

1. Open https://supabase.com
2. Click New Project
3. Enter project name
4. Choose password
5. Click Create Project

After you complete this step, come back and tell me:
"Step 1 completed"

Do NOT give me Step 2 yet.

Wait until I confirm Step 1 is completed.

---

## TROUBLESHOOTING

For every step provide:

Common Error 1:
Why it happens

How to fix it

Common Error 2:
Why it happens

How to fix it

---

## VALIDATION

Tell me exactly how to verify the step worked correctly.

Example:

You should now see:

* Project Dashboard
* Project URL
* Anon Key
* Service Role Key

If you can see these items then the step is complete.

---

## PROJECT TRACKER

Always show progress.

Example:

Project Progress:

✅ Step 1 Completed
⬜ Step 2 Pending
⬜ Step 3 Pending
⬜ Step 4 Pending

---

## IMPORTANT

Never give me 5, 10, or 20 steps at once.

Only give ONE next step.

Wait for my confirmation before continuing.

Assume I have zero technical knowledge.

Guide me until:

* Development is completed
* Database is completed
* Authentication is completed
* Brevo integration is completed
* HTML template uploads are completed
* Supabase Storage is completed
* Deployment to Vercel is completed
* Production testing is completed

Do not skip any step.

Do not assume anything is already configured.

Keep the process simple, sequential, and beginner-friendly.
